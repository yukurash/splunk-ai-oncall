# シナリオ3: cartFailure — 未再現（不採点）

- flag: `cartFailure` variant=on（flagd 反映確認済み）
- 結果: **観測窓内で cart に有意な APM 障害が出なかった**（cart err=1/health=Ok、checkout err=0）。

## なぜ出なかったか（正直な記録）
- demo v2.2.0 の `cartFailure` は cart の **`EmptyCart`（checkout 完了後に呼ばれる）限定**で失敗する低頻度パスとみられる。
- かつ checkout 側でそのエラーが致命扱いされず（注文自体は成立）、**APM のサービスエラーとして表面化しない**。
- 複数回・時間窓を変えて収集したが、cart/checkout とも rootCause エラー無し。

→ 捏造を避け、本シナリオは **不採点**とする（フラグ設計上 APM では観測しにくい例として、それ自体が知見）。

## 学び
- 「フラグを ON にした＝APM に必ず出る」ではない。**握りつぶされる/低頻度/非リクエスト系の障害は APM トレースに表れない**（#5 メモリ・#6 一部と同根）。観測性は「どの信号に出るか」を理解して設計する必要がある。
